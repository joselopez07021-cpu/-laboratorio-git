Git

Sistema de control de versiones distribuido para guardar y comparar cambios.

GitHub

Plataforma en la nube para alojar repositorios, colaborar y revisar codigo.

Equipo

Facilita ramas, fusion de cambios, historial claro y trabajo paralelo.

Conceptos clave

Lo esencial que debes conocer

Repositorio

Es la carpeta del proyecto con su historial de versiones, ramas y commits.

Commit

Es una captura de cambios con mensaje descriptivo para dejar evidencia del avance.

Branch

Permite trabajar nuevas ideas o funciones sin afectar la rama principal.

Pull Request

En GitHub sirve para proponer cambios, revisarlos y aprobarlos antes de unirlos.

Flujo recomendado

Proceso simple para trabajar de forma ordenada

1

Clonar o iniciar

Crea el repositorio local con git init o descarga uno existente con git clone.

2

Guardar cambios

Usa git add . y git commit para registrar avances claros y frecuentes.

3

Subir a GitHub

Con git push publicas tu progreso y mantienes respaldo remoto del proyecto.

4

Revisar y colaborar

Comparte ramas, abre pull requests y revisa codigo antes de mezclar cambios importantes.

Buenas practicas

Consejos para trabajar mejor

Haz commits pequenos

Es mas facil entender, revisar y revertir cambios cuando cada commit tiene un objetivo claro.

Escribe buenos mensajes

Un mensaje corto y preciso ayuda a comprender el historial del proyecto rapidamente.

Usa ramas por tarea

Separar funcionalidades evita conflictos y mantiene estable la rama principal.
